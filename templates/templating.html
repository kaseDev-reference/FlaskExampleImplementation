Templating and Rendering

Templates make it easy for the html that is sent to the browser to be manipulated and changed by code on the backend before it is parsed. Some useful features include the ability to extend an html page from another page and use python values and control flow.

html extending

{% raw %}

The header for this and all other pages on this site are a good example of the utility of html extensions. This is done through blocks denoted with {% block block_name %}{% endblock %} and {% extends "parent.html" %}. The parent html page will define the structure of the page and declare the blocks while the child html page will extend the parent and fill the named blocks with content.

Accessing variables and using control flow

Passing Python variables to the html page is a prime feature of Flask and Jinja. It is done by passing a named argument to render_template() and then using that name to reference the variable within html inside double curly braces like so {% raw %} {{ }} {% endraw %}.

As an example, the name {{ name }} comes from Python. And we can also deal with more complex data like lists and objects. The following list is a good example of this.

    {% for item in items %} {% if item.startswith('f') %}
  • {{ item }}
    • {% else %}
  • {{ item }}
    • {% endif %} {% endfor %}

The above was achieved by passing a list with 4 items from python to this html page, iterating through the list 1 item at a time, and then using an if statement to selectively color the item based on the first letter of the word.
